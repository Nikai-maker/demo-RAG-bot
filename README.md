# Demo RAG Telegram Bot

Production-style Retrieval-Augmented Generation system

## 📌 Описание проекта

Демонстрационный Telegram-бот, реализующий архитектуру RAG (Retrieval-Augmented Generation).

Система позволяет:
- Загружать текстовые файлы через Telegram
- Индексировать документы в векторную базу данных
- Выполнять семантический поиск
- Отвечать на вопросы на основе загруженного контента
- Поддерживать multi-tenant изоляцию пользователей

Проект демонстрирует архитектурный подход, близкий к production-решениям.

---

## 🏗 Архитектура

Система построена по принципу разделения ответственности и масштабируемости.

Telegram  
   ↓  
n8n (оркестрация)  
   ↓  
Redis (очередь задач)  
   ↓  
Python Worker (обработка)  
   ↓  
PostgreSQL + pgvector (хранение и поиск)

---

## 🔹 Компоненты

### 1. Telegram Bot

Интерфейс взаимодействия с пользователем:
- Принимает файлы
- Принимает вопросы
- Отправляет ответы
    

---

### 2. n8n (оркестратор)

Используется как workflow-движок.

Функции:
- Получение файлов из Telegram
- Сохранение файла в shared storage
- Постановка задач в Redis
- Отправка ответов пользователю
    

Почему n8n:
- Гибкая оркестрация
- Лёгкая модификация процессов
- Подходит для enterprise automation

---

### 3. Redis

Используется как очередь задач.

Назначение:
- Асинхронная обработка документов
- Разделение оркестрации и бизнес-логики
- Возможность горизонтального масштабирования воркеров

---

### 4. Python Worker (FastAPI)

Отдельный контейнер, выполняющий обработку документов.

Функции:
- Извлечение текста
- Чанкинг
- Генерация embeddings
- Сохранение в базу данных
- Семантический поиск

Worker можно масштабировать горизонтально.

---

### 5. PostgreSQL + pgvector

Хранилище данных и векторный поиск.

Содержит:
- Tenants (пользователи)
- Documents (метаданные)
- Document Chunks (векторизированные фрагменты текста)

Используется индекс ivfflat для ускорения поиска.

---

## 🔐 Multi-Tenant Логика

Каждый пользователь Telegram = отдельный tenant.

Это обеспечивает:
- Изоляцию данных
- Независимый поиск
- Масштабируемость системы

## 🔎 Пример сценария работы

1. Пользователь загружает файл в Telegram
2. n8n сохраняет файл
3. Задача отправляется в Redis
4. Worker обрабатывает документ
5. Документ индексируется в pgvector
6. Пользователь задаёт вопрос
7. Выполняется векторный поиск
8. Возвращается ответ
