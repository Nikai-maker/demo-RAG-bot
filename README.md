# Сервис (пет-проект) - RAG бот

Этот проект представляет собой Proof-of-Concept RAG-бота, который работает через Telegram, обрабатывает PDF-файлы и отвечает на вопросы с использованием LLM и векторных эмбеддингов.

## Архитектура

Система состоит из следующих компонентов:

- **Telegram Бот** — пользовательский интерфейс
- **n8n** — оркестрация рабочих процессов
- **RabbitMQ** — брокер сообщений
- **Python Worker** — обработка документов, генерация эмбеддингов, ответы на запросы с использованием LLM (OpenAI)
- **PostgreSQL с pgvector** — хранение векторных эмбеддингов и метаданных

## Требования

- Docker и Docker Compose
- Telegram Bot Token
- OpenAI API Key

## Запуск проекта

1. Склонируйте репозиторий
2. Создайте файл `.env` на основе `.env.example` (или отредактируйте существующий)
3. Укажите свои значения для `TELEGRAM_BOT_TOKEN` и `OPENAI_API_KEY`
4. Запустите сервисы:

```bash
docker-compose up --build
```

Сервисы будут доступны по следующим адресам:

- **n8n**: http://localhost:5678
- **RabbitMQ Web UI**: http://localhost:15672 (логин: `guest`, пароль: `guest`)
- **PostgreSQL**: `localhost:5432`, база `db`, пользователь `admin`

## Как использовать

1. Запустите бота в Telegram
2. Отправьте PDF-файл
3. Задайте вопрос по содержанию файла
4. Бот вернёт ответ, основанный на данных из документа

## Структура проекта

- `n8n/` — данные и воркфлоу n8n
- `worker/` — Python-обработчик с логикой RAG
- `postgres/` — инициализация и данные PostgreSQL
- `rabbitmq_data/` — данные RabbitMQ
- `docker-compose.yml` — оркестрация сервисов
- `.env` — переменные окружения
